# 文档索引

> 版本基线：2026-07-16 完整长线系统版。系统实现状态以“当前原型”文档为准；高关数值平衡仍只以实机验收记录为证据。

## 阅读顺序与冲突优先级

1. `当前原型策划.md`：确定本轮开发范围和版本验收口径。
2. `战斗与关卡循环策划.md`、`经济与成长策划.md`、`配件与词条系统策划.md`：确定当前系统规则。
3. `UI与交互重构策划.md`：确定基地、战斗、升级和结算的界面行为。
4. `关卡调参与试玩记录.md`：只记录真实样本和未完成验证，不用推测数据覆盖实测。
5. `R2实机验收与高关采样.md`：R2 实机证据、固定构筑和后续三局制采样口径。
6. `远期总策划.md`、`远期装备系统策划.md`：完整系统规则与内容目录；与实机平衡记录冲突时，以有证据的验收结果为准。

## 游戏策划

- `当前原型策划.md`：当前原型策划说明，记录已经落地的系统、已移除入口和短期迭代优先级。
- `战斗与关卡循环策划.md`：当前战斗、关卡、技能、奖励和战斗 HUD 的专项策划。
- `配件与词条系统策划.md`：当前配件实例、主词条、副词条、强化、重铸、背包管理和推荐逻辑专项策划。
- `经济与成长策划.md`：当前资源、成长循环、已移除基地永久强化入口和后续经济设计建议。
- `UI与交互重构策划.md`：当前基地、战斗 HUD、局内升级、结算、响应式和无障碍的实现规格与验收标准。
- `R2实机验收与高关采样.md`：R2 两档实机结果、UI 注入自动化、固定构筑夹具、三局制记录表和经济调参门槛。
- `远期总策划.md`：远期完整游戏总策划，包含产品定位、核心循环、战斗、关卡、角色、怪物、Boss、奖励、经济、任务、解锁和开发优先级。
- `远期装备系统策划.md`：远期装备系统专项策划，包含武器、品质、强化、枪口、弹匣、瞄具、枪管、枪托、弹芯、模块、芯片、词条、套装、流派和掉落。

## 技术架构

- `技术架构.md`：当前 Nuxt 3 SPA 原型架构，以及后续演进到 Nitro API、PostgreSQL/Neon、JWT、Vercel Serverless 的部署方案。

## 当前重构里程碑

| 里程碑 | 范围 | 完成标准 |
| --- | --- | --- |
| R1 数据可信 | Boss 预览、存档迁移、背包溢出、失败结算 | 展示与实际公式同源；无静默资源损失 |
| R2 核心界面与经济闭环 | 基地首屏、战斗 HUD、升级三选一、结算、收藏保护、三资源重铸、锁定副词条 | 两种目标分辨率均能完成核心流程；金币/合金具备重复消耗口；收藏实例不被出售或自动回收 |
| R3 平衡扩样 | 第 20、40、60、100 关 | 每个节点固定构筑连续 3 局并记录证据 |
| R4 长线成长 | 金币/合金消耗、伤害成长、关卡解锁 | 资源形成闭环后再扩展 100 关以上内容 |
