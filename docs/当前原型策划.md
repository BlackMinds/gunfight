# 当前原型策划说明

## 1. 文档定位

本文记录当前 Nuxt 原型已经落地的游戏内容，作为短期迭代和验收基准。

2026-07-15 起进入 R1/R2 重构：先修复数据可信、存档和背包安全，再优化基地与战斗界面。当前只把第 1、5、10 关视为有正式试玩证据的平衡区间；第 11～10000 关均不得表述为“已经平衡”。

远期设想仍可参考：

- `远期总策划.md`
- `远期装备系统策划.md`

但当前开发、测试和界面调整，应优先以本文和拆分出的专项文档为准。

## 2. 当前版本核心体验

当前原型是一个俯视角自动射击 RPG：

1. 玩家在基地选择目标关卡。
2. 进入战斗后，角色自动攻击最近敌人。
3. 玩家负责移动、躲避包围和使用 3 个主动技能。
4. 关卡结算获得金币、经验、合金、零件和配件掉落。
5. 回到基地整理配件背包，选择更好的词条实例继续推进。

当前版本的长期目标不再是基地永久强化，而是围绕配件实例、词条底子、强化和重铸形成刷装循环。

## 4. 当前主界面结构

### 4.1 基地页

基地页用于关卡准备和配件整理。重构后的优先级不是平均分配空间，而是先完成“选关—看风险—部署”，再进入配件管理：

- 当前战力与下一关敌情。
- 目标关卡调整。
- 奖励预览。
- 部署按钮。
- 配件背包。
- 配件详情面板。

在 1280×720 下，目标关卡、奖励预览和部署按钮必须首屏可见；在 390×844 下，部署操作必须先于大面积角色展示和背包列表。

基地页不再展示“枪械校准 / 护甲整备 / 磁吸轨道”三张永久强化卡。

### 4.2 战斗页

战斗页使用暗金战术 HUD，主要区域包括：

- 左侧：训练者状态与基础属性。
- 顶部：合并后的波次、击杀和本关用时。
- 右侧：实时 DPS 与必要战斗摘要；详细构筑贡献留到结算。
- 底部：3 个主动技能。

桌面端显示移动方式和 `1/2/3` 技能键位；移动端使用不小于 44×44 CSS 像素的触控目标。

### 4.3 结算页

结算页展示：

- 胜负结果。
- 本次资源奖励。
- 掉落配件。
- 配件对比和立即装备入口。
- 战后选择：修复生命、拆解零件。

失败局不参与 45～90 秒完整通关时长评价，改为展示失败波次、剩余敌人和死亡组合。

## 5. 当前版本设计原则

1. 配件是当前局外成长核心。
2. 基地页优先服务“选关”和“整理配件”，不再堆叠永久强化入口。
3. 同名配件要因为 roll、主词条、副词条和强化等级产生差异。
4. 推荐装备逻辑只提供建议，不替玩家自动决定。
5. 战斗中锁定背包，换装只能在基地或结算页完成。
6. 展示数据和实际战斗必须调用同一套共享公式。
7. 不允许通过容量截断、存档迁移或失败结算文案造成隐性损失与误导。

## 7. 本轮完成定义

- `npm test`、`npx nuxi typecheck`、`npm run build` 全部通过。
- 第 1、5、10 关既有实测结论不因 UI 重构发生倒退。
- Boss 预览误差不超过 1%，背包满时不静默丢失物品。
- 连续注入超过 24 件配件时实例守恒；收藏实例不进入批量出售或自动回收结果。
- 重铸必须同时校验零件与金币，锁定副词条时额外校验合金，资源不足时不得部分扣除。
- 1280×720 和 390×844 能完成基地部署、战斗升级和结算返回流程。
- 所有未由真人手操完成的高关与纯移动反制仍明确标注“待人工确认”。

## 8. 2026-07-16 長線系統首版邊界

本輪開始落地遠期方向，但不代表遠期文檔已全部完成：

- 多武器首版提供突擊步槍、風暴衝鋒槍、餘燼霰彈槍、霜痕狙擊槍，並實際改變傷害、射速、射程、穿透、彈丸數與元素。
- 元素首版提供灼燒、感電、冰緩；其他傷害類型與異常狀態仍待後續戰鬥壓力驗證。
- 天賦首版每個分支提供 1 個可升級節點；套裝先接入 2 件數值效果與 2/4 件進度展示，4 件機制效果尚未封板。
- 離線收益按最高通關關卡計算，最多累積 8 小時；每日任務先提供擊殺、通關與重鑄三類。
- 傳說/神話已進入品質、詞條數、強化上限、篩選與高關掉落資料層，但不提前投入 3000 / 7000 關以前的正式掉落池。
- 雲端存檔只完成資料邊界與界面狀態說明；真實同步仍依賴帳號、API、資料庫和衝突合併規則。
