# 当前原型策划说明

## 1. 文档定位

本文记录当前 Nuxt 原型已经落地的游戏内容，作为短期迭代和验收基准。

2026-07-15 起完成 R1/R2 数据可信与界面重构；2026-07-16 完成全局一致性修复和 R3 重新采样；2026-07-18 完成 R4 自动验收；2026-07-21 完成 R5 第 501～10000 关开发层、24 局自动验收，以及项目负责人授权的 R4/R5 共 9 局 Codex 正常时速发布实测。该证据用于解除本项目发布门槛，但仍如实标注为 Codex 实测，不冒充外部真人主观签字。

## 2. 当前版本核心体验

当前原型是一个俯视角自动射击 RPG：

1. 玩家在基地选择目标关卡。
2. 进入战斗后，角色自动攻击最近敌人。
3. 玩家负责移动、躲避包围和使用 3 个主动技能。
4. 关卡结算获得金币、经验、合金、零件和配件掉落。
5. 回到基地整理配件背包，选择更好的词条实例继续推进。

当前版本的长期目标不再是基地永久强化，而是围绕配件实例、词条底子、强化和重铸形成刷装循环。

## 4. 当前主界面结构

### 4.1 基地页

基地页用于关卡准备和配件整理。重构后的优先级不是平均分配空间，而是先完成“选关—看风险—部署”，再进入配件管理：

- 当前战力与下一关敌情。
- 目标关卡调整。
- 奖励预览。
- 部署按钮。
- 配件背包。
- 已装备配件与直接培养入口。
- 武器、天赋、套装、任务等长期养成。

在 1280×720 下，目标关卡、奖励预览和部署按钮必须首屏可见；在 390×844 下，部署操作必须先于大面积角色展示和背包列表。

基地页不再展示“枪械校准 / 护甲整备 / 磁吸轨道”三张永久强化卡。

### 4.2 战斗页

战斗页使用暗金战术 HUD，主要区域包括：

- 左侧：训练者状态与基础属性。
- 顶部：合并后的波次、击杀和本关用时。
- 右侧：实时 DPS 与必要战斗摘要；详细构筑贡献留到结算。
- 底部：3 个主动技能。

桌面端在首次进入或静止 3 秒后提示移动方式，并显示 `1/2/3` 技能键位；移动端使用 136×136 虚拟摇杆和不小于 72×72 CSS 像素的技能触控目标。

### 4.3 结算页

结算页展示：

- 胜负结果。
- 本次资源奖励。
- 掉落配件。
- 配件对比和立即装备入口。
- 单次战后处置：修复生命或拆解零件；同一结算只能选择一次。选择修复后直接进入下一关会保留修复后的生命，返回基地后重新部署仍恢复满生命。

失败局不参与 45～90 秒完整通关时长评价，改为展示失败波次、剩余敌人和死亡组合。

## 5. 当前版本设计原则

1. 配件是当前局外成长核心。
2. 基地页优先服务“选关”和“整理配件”，不再堆叠永久强化入口。
3. 同名配件要因为 roll、主词条、副词条和强化等级产生差异。
4. 推荐装备逻辑只提供建议，不替玩家自动决定。
5. 战斗中锁定背包，换装只能在基地或结算页完成。
6. 展示数据和实际战斗必须调用同一套共享公式。
7. 不允许通过容量截断、存档迁移或失败结算文案造成隐性损失与误导。

## 7. 本轮完成定义

- `npm test`（22 个测试文件、116 条用例）、`npx vue-tsc --noEmit`、`npm run build` 全部通过。
- 第 1、5、10 关既有实测结论不因 UI 重构发生倒退。
- Boss 预览误差不超过 1%，背包满时不静默丢失物品。
- 连续注入超过 24 件配件时实例守恒；收藏实例不进入批量出售或自动回收结果。
- 重铸必须同时校验零件与金币，锁定副词条时额外校验合金，资源不足时不得部分扣除。
- 1280×720 和 390×844 能完成基地部署、战斗升级和结算返回流程。
- 390×844 顶部操作条、波次指挥条与角色状态卡互不重叠；已装备配件可直接强化、锁词条和重铸。
- 第 20、40、60、100 关新固定构筑连续 3 局均通关，完整通关时间全部落在 45～90 秒。
- R4/R5 高关发布证据明确标注为项目负责人授权的 Codex 正常时速实测，不冒充外部真人主观签字；仍未覆盖的纯移动无伤等专项继续标注“待人工确认”。

## 8. 云存档生产化与部署边界

2026-07-22 完成本地与模拟可靠性收口：并发重复注册由数据库唯一约束稳定返回 409；数据库首次初始化失败后可重试；JWT 配置、凭证、`baseRevision`、payload 类型和存档核心字段均在服务端校验；单份存档按 UTF-8 字节限制为 1 MB；写入在用户级事务锁内比较修订号，异常必须回滚并统一返回不泄露数据库细节的 503。客户端写入进入单一队列，连续保存会等待前一次返回新修订号后再发送。

本轮没有 Vercel/Neon 线上授权，因此只确认本地与模拟测试、类型检查和 Vercel 生产构建通过。Preview/Production 的真实连接、环境隔离、日志、双会话冲突和恢复能力必须按 `Vercel与Neon部署检查表.md` 另行验收；完成前不得宣称云存档已通过线上生产验收。
