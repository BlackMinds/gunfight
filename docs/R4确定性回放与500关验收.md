# R4 确定性回放与第 500 关验收

## 1. 验收边界

本轮只验证第 100～500 关开发内容，不修改正式开放范围。`PUBLISHED_STAGE_CAP` 继续保持 `100`；在真人长时间走位手感完成前，不启动“是否把正式上限提升到 500”的独立评审。

回放固定条件：

- 节点：100 / 150 / 200 / 250 / 300 / 350 / 400 / 450 / 500。
- 每节点连续 3 个有效样本，共 27 个有效样本。
- 固定步长 `1/60` 秒，基础种子 `0x6a18`；每局重新注入所属区段的固定构筑。
- 页面隐藏、窗口失焦、人工按键、画布不可用或帧输入中断均标记为无效，使用相同节点重新尝试，不以估算值补齐。
- 正式批次结果为 `27/27` 有效、`0` 局剔除；另一次视口重连后的受污染尝试未进入本表。

## 4. 调参决策

九个节点的中位数均在 45～90 秒目标内，通关率均为 3/3，没有死亡组合。不存在两个相邻节点同方向越界，因此：

- 不调整全局生命、伤害或速度曲线。
- 不调整敌人密度、组合或预警时间。
- 保留“预警 → 组合 → 密度 → 生命 / 伤害”的调参优先级；后续只有 `adjacentDurationBreaches` 返回相邻同向越界证据时，才允许进入全局曲线评审。

## 5. 双视口与走位验收

### 1280×720

- 战场、波次条、角色信息、实时输出和技能区均在单屏内，无横向遮挡。
- 爆区半径、精英外圈和协同范围使用独立颜色与虚线节奏；桌面尺寸继续使用原绘制强度。

### 390×844

- 虚拟摇杆和三个 72×72 技能按钮保持在视口内，拖拽摇杆及 1 / 2 / 3 技能输入能改变战斗状态。
- 修复回放完成状态条遮挡移动端 HUD：隐藏结果节点继续保留，完成态可视面板自动收起。
- 小视口下提高爆区填充和描边强度、协同圈线宽，并给精英双词缀标签增加高对比底板与词缀色边框。

### 真人门槛

浏览器连续控制只能证明输入链路与布局可用，不能冒充真人对疲劳、反应、触屏误触和长时间走位手感的判断。仍需真人在第 350、450、500 关各完成至少 1 局，并记录：

1. 爆区是否能在 0.9 秒内辨认并离开。
2. 双词缀标签是否能在交战中快速读出。
3. 协同范围是否能支持“先杀精英”的即时决策。
4. 390×844 下摇杆与技能是否出现误触、手指遮挡或疲劳。

在这 3 局真人记录补齐前，本项状态为“自动化与连续控制通过，真人手感待验收”。

## 6. 正式上限结论

R4 的固定构筑、27 局回放、遥测、调参门槛和双视口自动验收已经通过；真人长时间走位手感仍未完成。因此当前不启动 500 上限独立评审，正式硬上限明确维持第 100 关。
