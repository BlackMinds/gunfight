# UI 与交互重构策划

## 1. 文档定位

本文是 2026-07-15 起基地、战斗 HUD、局内升级和结算页的实现规格。目标不是更换现有暗金军械风格，而是在保留视觉识别度的前提下，提高信息可信度、首屏任务完成率、响应式体验和无障碍可用性。

## 2. 体验原则

1. 玩家首先看到当前任务和下一步操作，其次才是装饰和详细数据。
2. 所有敌情、奖励和属性预览必须与实际战斗共用数据源。
3. 战斗 HUD 只保留会影响即时决策的信息，复盘数据留到结算。
4. 失败结算描述失败进度，不用完整通关时长标准评价提前阵亡。
5. 背包满、资源不足、功能不可用必须给出明确状态，不静默丢弃或伪装成可用。
6. 桌面端与移动端使用同一信息层级；移动端改变布局，不删除关键操作。

## 3. 基地信息架构

### 3.1 首屏任务流

基地首屏按以下顺序组织：

1. 当前战力与构筑摘要。
2. 目标关卡、关卡类型、敌人预览。
3. 预计奖励与风险提示。
4. 主操作“立即部署”。
5. 装备与背包管理。
6. 武器、天赋、套装、任务等长期养成。

“立即部署”在桌面端保持右侧任务卡底部可见；移动端使用底部安全区吸附操作条。装备详情和背包可滚动，但不能让主操作依赖嵌套滚动才能出现。

### 3.2 关卡选择

- 正式模式只允许选择 `1` 到 `最高已通关 + 1`。
- 开发环境可显示“调试选关”，范围保留到 10000；基地与战斗 HUD 显示 R5 战区、词缀数和 Boss 阶段，并明确正式版本仍封顶第 100 关。
- 首领、精英、资源规则重叠时，显示复合标签，例如“首领关 · 零件加成”。
- Boss 生命、伤害和预警摘要直接读取 `shared/game/waves.ts` 与 `shared/game/formulas.ts`。

### 3.3 背包状态

- 显示 `已用 / 容量`。
- 达到 80% 容量时显示预警。
- 满包掉落进入临时战利品区或明确自动回收；任何配件都不得因为数组截断而静默消失。
- 装备比较至少展示输出、生存、功能三个维度及关键属性差值。

### 3.4 收藏与重铸

- 配件卡以星标显示收藏保护，并提供“收藏”筛选。
- 出售模式中收藏卡显示锁定态，不能勾选；全选只作用于未收藏实例。
- 每条副词条提供不小于 44×44 CSS 像素的锁定操作，一次最多锁定 1 条。
- 重铸按钮同时展示零件、金币以及可选的合金成本；资源不足时显示具体缺口并禁用操作。
- 受保护实例导致背包超过容量时，基地与结算的继续部署按钮禁用，并显示整理指引。
- 已装备配件卡提供“培养”入口，可直接强化、锁定副词条和重铸，不要求先卸回背包。

## 4. 战斗 HUD

### 4.1 信息层级

| 层级 | 信息 | 要求 |
| --- | --- | --- |
| 一级 | 生命、Boss 阶段、危险预警、技能冷却 | 始终可见，不能被横幅遮挡 |
| 二级 | 波次、击杀、时间、当前 DPS | 合并重复信息，单行优先 |
| 三级 | 总伤害、历史峰值、装备贡献 | 可收起或仅在结算展示 |

### 4.2 控制提示

- 桌面端首次进入或 3 秒内无移动时提示 `WASD / 方向键移动`。
- 三个技能分别显示 `1`、`2`、`3` 键位。
- 技能按钮显示名称、用途、剩余冷却；图标 `alt` 不重复朗读可见文字。
- 移动端预留虚拟摇杆和大尺寸技能触控区，最小点击目标 44×44 CSS 像素。
- 当前实装为左下 136×136 虚拟摇杆与右下三枚 72×72 技能按钮；移动端战斗固定为单屏，不依赖页面滚动寻找操作。

### 4.3 Boss 预警

- 危险区至少使用颜色、轮廓/纹理、倒计时或阶段文字中的两种，不只依赖红色。
- 第三阶段预警继续保留当前 0.72 秒基线，本轮不继续削弱。
- “冲刺冷却时纯方向移动能否稳定无伤”仍是人工验收项，不能写成已通过。

## 5. 局内升级

升级卡必须展示：

- 强化名称与一句用途说明。
- 当前值 → 强化后值。
- 构筑标签：输出、穿透、机动、拾取或生存。
- `1 / 2 / 3` 快捷键。

三项强化不能只凭描述判断强弱。需要记录选择率、选择后的通关率、逐波时间和贡献值。任何单项在同一验证区间选择率持续高于 70%，且通关率不低于其他项时，进入平衡复核。

## 6. 结算页

### 6.1 胜利

依次展示：胜利结果、核心奖励、高价值掉落、时间评价、构筑复盘、下一关/返回基地。

### 6.2 失败

依次展示：失败波次、剩余敌人、死亡组合、受伤次数、构筑贡献和重新挑战。

失败局的时长区域固定显示“未完成，不计入通关时长目标”；不得显示“偏快/达标/偏慢”。

### 6.3 文案与本地化

- 玩家可见敌人名称统一使用中文，不显示 `grunt / ranged / heavy` 等内部 key。
- 主按钮使用动词，例如“重新挑战”“返回基地整备”。
- 暂未开放的设置入口使用禁用态和“开发中”说明，不能表现为可完成操作。

## 7. 响应式规范

### 7.1 桌面端 1280×720

- 基地的关卡选择、奖励和部署按钮首屏可见。
- 战斗 HUD 不遮挡中央主要移动区域。
- 结算主操作无需滚动即可看到。

### 7.2 移动端 390×844

- 页面只允许一个主滚动容器。
- 基地优先显示任务和部署，角色展示折叠或缩小。
- 底部操作考虑 `env(safe-area-inset-bottom)`。
- 不出现横向页面滚动。
- 顶部操作条、波次指挥条和角色状态卡不得发生像素重叠，三者相邻边界至少保留 8 CSS 像素。

## 8. 无障碍与状态规范

- 正文建议不低于 14px；12px 只用于非关键辅助信息。
- 文字与背景达到可辨识对比度，状态不能只用颜色区分。
- 焦点态必须可见，键盘可完成选关、升级选择、技能触发和结算操作。
- 按钮提供 normal、hover、focus、pressed、disabled 状态。
- 动效尊重 `prefers-reduced-motion`。

## 9. 验收标准

- [x] Boss 预览与实际生成参数误差不超过 1%。
- [x] 1280×720 基地首屏可看到并操作部署按钮。
- [x] 390×844 无横向滚动和嵌套滚动死区。
- [x] 背包满时没有配件静默丢失，并能看到处理结果。
- [x] 失败结算不再显示通关时长快慢评价。
- [x] 战斗 HUD 不重复展示两套波次状态。
- [x] 波次、击杀和时间合并到顶部指挥条；左侧不重复击杀，右侧仅保留实时 DPS。
- [x] 技能和局内升级均显示键位与明确数值变化。
- [x] 玩家可见区域不出现内部英文敌人 key。
- [x] 收藏实例不能被批量出售或自动回收，收藏状态随存档保留。
- [x] 重铸完整展示并校验零件、金币、合金；锁定副词条后保留值不变。
- [x] 移动端虚拟摇杆可驱动与键盘同源的移动向量，技能按钮均不小于 72×72。
- [x] 配件详情固定展示输出、生存、功能三维差值和传说/神话专属效果。
- [x] 基地任务流按部署、背包、长期养成排列，已装备配件可直接进入培养。
- [x] 390×844 顶部操作条、波次指挥条和角色状态卡互不重叠。
