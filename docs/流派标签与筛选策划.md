# 流派标签与筛选策划

## 1. 目标与边界

本系统把配件现有主词条、副词条和专属效果解释为稳定、可筛选的构筑标签，帮助玩家在 24 格背包内快速找到同一用途的配件。标签只解释配件倾向，不取代输出、生存、功能三维对比，也不自动装备或出售。

本批次固定四类标签：

| 标签 | 主要词条 | 玩家决策 |
| --- | --- | --- |
| 输出 | 伤害、射速、暴击率 | 提升直接火力 |
| 生存 | 最大生命、移速 | 提升承伤和走位容错 |
| 经济 | 拾取、经验 | 提升刷取和成长效率 |
| 穿透 | 穿透，或明确强化穿透的专属效果 | 处理重装与密集直线敌群 |

## 2. 标签生成规则

1. 汇总配件当前生效的主词条和副词条，不根据名称猜测。
2. 每件配件至少获得一个主标签；存在第二个有效方向时最多展示两个标签。
3. 穿透值大于 0 时固定获得“穿透”，避免被其他高权重词条覆盖。
4. 其余标签按归一化权重排序：输出取伤害、射速、暴击；生存取生命、移速；经济取拾取、经验。
5. 同分时使用固定优先级 `穿透 → 输出 → 生存 → 经济`，保证存档重载和测试结果稳定。
6. 强化、重铸后立即重新计算；标签不是存档字段，避免旧存档迁移和派生数据漂移。

## 3. 筛选与展示

- 背包增加“流派”筛选：全部、输出、生存、经济、穿透。
- 流派筛选与现有槽位、品质、收藏/推荐等筛选叠加，采用 AND 规则。
- 配件卡、详情和已装备培养区显示标签；颜色不能成为唯一识别方式，必须保留文字。
- 当前筛选无结果时继续使用既有空状态，不自动清除其他筛选条件。
- 出售模式仍受收藏保护约束；流派筛选只缩小选择范围，不改变返还和保护规则。

## 4. 逐件回收预览

- 出售模式下，每个未收藏配件卡显示该实例的金币和零件返还。
- 顶部继续显示所选实例总计；逐件值与总计必须调用同一个 `getAttachmentRecycleValue` 数据源。
- 收藏实例显示“收藏保护”，不显示为可出售状态。
- 强化等级带来的额外零件必须反映在逐件预览与总计中。

## 5. 验收标准

- [x] 四类流派标签由纯函数生成，重载和重复计算结果一致。
- [x] 穿透配件始终带“穿透”标签；纯生命/移速配件不误标为输出。
- [x] 流派、槽位、品质和状态筛选可以叠加。
- [x] 强化或重铸后标签立即更新，不新增存档字段。
- [x] 出售模式逐件返还之和等于顶部总计。
- [x] 收藏配件仍不能出售，流派筛选不绕过保护。
- [x] 1280×720 与 390×844 下标签和返还预览不造成横向溢出。

## 6. 实现与验证记录（2026-07-23）

- 标签与逐件返还已接入配件卡，筛选控件使用 `data-testid="build-tag-filter"`，逐件返还使用 `data-testid="item-recycle-preview"`。
- 标签与返还分别复用 `buildAttachmentTags` 和 `getAttachmentRecycleValue`，没有新增派生存档字段。
- `attachment-domain`、`r2-economy` 与全量测试通过；1280×720、390×844 实际界面均无横向溢出。
