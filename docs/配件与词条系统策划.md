# 配件与词条系统策划

## 1. 系统定位

当前版本的核心局外成长是配件实例。

玩家追求的不是单纯获得某个配件模板，而是获得：

- 更高品质。
- 更高 roll 值。
- 更合适主词条。
- 更好的副词条组合。
- 值得强化或重铸的底子。

2026-07-15 重构同时要求“可理解”和“不会丢”：属性名称必须对应真实机制，整数属性不能制造虚假的小数 roll 差异，背包溢出必须有明确处理结果。

## 2. 配件槽位

当前支持 8 个槽位：

| 槽位 | 当前定位 |
| --- | --- |
| 枪口 | 输出向，偏伤害和射速。 |
| 弹匣 | 射速、装填感和输出节奏。 |
| 瞄具 | 命中、射速、暴击倾向。 |
| 枪管 | 穿透和弹道强化。 |
| 枪托 | 移速、生存和稳定性。 |
| 弹芯 | 伤害、穿透、特殊弹药方向。 |
| 模块 | 拾取、生存、机制类属性。 |
| 芯片 | 经验、收益、构筑辅助。 |

同槽位只能装备 1 件配件。

## 4. 配件实例

每次掉落都会生成独立实例。

实例字段包括：

- 实例编号。
- 模板名。
- 品质。
- roll 值。
- 强化等级。
- 主词条。
- 副词条列表。
- 汇总后的实际属性。

同名配件可以因为实例 roll 和副词条不同而价值不同。

## 5. 品质与副词条数量

当前品质分为：

| 品质 | 副词条数量 | 当前定位 |
| --- | --- | --- |
| 普通 | 1 | 过渡、分解或低成本重铸测试。 |
| 精良 | 2 | 前期可用。 |
| 稀有 | 3 | 当前主要追求品质。 |
| 史诗 | 4 | 当前高价值底子。 |

品质也会影响主属性数值缩放和副词条数值区间。

穿透是整数属性。生成、强化和界面展示均以整数档位处理，不把小数 roll 四舍五入后伪装成有差异的实例；穿透底子的价值主要来自品质、副词条和强化上限。

## 6. 词条类型

当前已接入的词条 key：

| 词条 | 表现 | 参与战斗 |
| --- | --- | --- |
| 伤害 | 百分比提高子弹伤害 | 是 |
| 射速 | 百分比提高开火频率 | 是 |
| 最大生命 | 提高玩家最大生命 | 是 |
| 拾取 | 提高金币和经验拾取半径 | 是 |
| 移速 | 提高玩家移动速度 | 是 |
| 穿透 | 增加子弹穿透次数 | 是 |
| 经验 | 提高经验获得 | 是 |
| 暴击率 | 子弹有概率造成更高伤害 | 是 |

命名必须与机制一致：当前只提高最大生命的“吸血模块”在 R1 更名为“生命模块”；旧存档名称通过迁移兼容。真正的吸血词条需要实现按实际伤害恢复生命、每秒恢复上限和遥测后才能重新使用“吸血”名称。

## 7. 主词条与副词条

### 7.1 主词条

主词条来自配件模板基础属性。

例如：

- 聚束枪口的主词条是伤害。
- 快速弹匣的主词条是射速。
- 穿甲枪管的主词条是穿透。

强化当前只提升主词条。

### 7.2 副词条

副词条随机生成，不能与主词条重复。

副词条用于让同名配件形成差异，例如：

- 伤害主词条 + 暴击率副词条。
- 射速主词条 + 拾取副词条。
- 经验主词条 + 移速副词条。

## 8. 强化

当前强化规则：

- 只对选中配件生效。
- 只能在非战斗状态操作。
- 消耗零件。
- 提升主词条数值。
- 强化等级有品质上限。

当前强化上限：

| 品质 | 强化上限 |
| --- | --- |
| 普通 | +3 |
| 精良 | +5 |
| 稀有 | +7 |
| 史诗 | +10 |

## 9. 重铸

当前重铸规则：

- 只刷新副词条。
- 保留配件模板、品质、主词条和强化等级。
- 同时消耗零件和金币。
- 重铸后重新计算配件总属性。
- 最多锁定 1 条副词条；执行重铸时额外消耗合金，被锁词条的类型与数值原样保留。
- 锁定选择本身不预扣资源；取消锁定不产生损耗。
- 任一资源不足时禁用重铸，并明确显示缺少的零件、金币或合金数量。

| 品质 | 标准零件成本 | 金币成本 | 锁定合金成本 |
| --- | ---: | ---: | ---: |
| 普通 | 2 | 40 | 1 |
| 精良 | 5 | 70 | 1 |
| 稀有 | 7 | 110 | 2 |
| 史诗 | 10 | 160 | 3 |

## 10. 背包管理

当前背包功能：

- 最近获得、品质优先、槽位整理排序。
- 全部、推荐、可替换、低品质筛选。
- 按槽位筛选。
- 清理低品质。
- 收藏筛选与收藏保护。
- 选中后展示配件详情。
- 装备后同槽旧配件回到背包。

容量与安全规则：

- R1 基础容量为 24 件，界面始终显示占用。
- 达到 80% 容量时提醒整理。
- 不允许通过 `.slice()` 或其他数组截断静默删除配件。
- 溢出自动回收时，优先普通、低 roll、低强化且未收藏的物品；本次高品质掉落和换下装备受保护。
- 结算页展示所有自动回收物和返还资源。
- 收藏实例不能进入出售选择或自动回收候选；若受保护实例超过容量，则保留全部实例并阻止继续部署。

战斗中背包锁定，只能查看少量预览，不允许装备、强化或重铸。

## 11. 推荐装备逻辑

当前推荐装备根据新旧同槽配件比较给出判断。

主要参考：

- 品质差异。
- 输出收益：伤害、射速、暴击率、穿透。
- 生存收益：最大生命、移速。
- 功能收益：拾取、经验。

评分只用于解释差异，不把所有构筑压缩成一个总分。详情面板必须至少给出输出、生存、功能三个分项，并列出造成推荐的主要属性变化。

推荐结果只作为提示，不自动替换玩家装备。

## 13. 验收标准

- [x] 连续获得超过容量的掉落时，没有实例无提示消失。
- [x] 换装后旧配件仍在背包或明确进入回收结果。
- [x] 收藏配件不会被批量清理或自动回收。
- [x] 重铸同时扣除零件和金币；锁定时额外扣除合金并原样保留目标副词条。
- [ ] 穿透属性始终显示整数，且实际穿透次数与界面一致。
- [ ] “生命模块”的名称、说明和实际最大生命增益一致。
- [ ] 推荐详情能解释至少一个输出、生存或功能属性差值。
